# clients

Outbound integrations: NATS, HTTP, declarative apimap, Redis, cache, webhooks, natsmap.

## `clients/nats/`

Typed NATS / JetStream client wrapper. `natsclient.Connect(ctx, Config) (*Client, error)` opens a connection + JetStream context. Generic `Publisher[T]` / `Subscribe[T]` over an opt-in `Codec` (JSON default). Auto-ack handler model: handler returns nil → Ack, err → Nak with exponential backoff, decode-fail → Term (poison pill). `MaxInFlight` semaphore caps handler concurrency. Idempotent `EnsureStream(ctx, StreamConfig)` for app-managed stream lifecycle. Errors map to `*errs.Error` with stable Code constants (`connect_failed`, `stream_not_found`, `publish_failed`, …). Opt-in slog/Prometheus observability via `WithLogger` / `WithMetrics`.

**Handler resilience opts:** `WithErrorClassifier(func(error) AckAction)` for declarative Ack/Nak/Term routing (validation errors → Term, transient → Nak); panic-recovery built into `dispatchRaw` (slot released, Nak issued, Warn-logged) with optional `WithPanicHandler(func(any))` callback; `WithAckProgress(d)` auto-heartbeats `InProgress()` so long handlers survive AckWait; `Msg[T].InProgress()` / `RawMsg.InProgress()` is the manual escape hatch.

**Pull-mode:** `NewPullSubscription[T](c, subject, opts...) + .Fetch(ctx, batchSize, maxWait) []*PendingMsg[T] / .Run(ctx, handler, batchSize, maxWait) / .Drain()` — typed pull for cron-style/batch consumers. Decode failures auto-Term'd as poison-pills inside Fetch; remaining decodes pass through. Requires `WithDurable`.

**TLS:** `WithTLSConfig(*tls.Config)` verbatim OR `WithRootCAs(*x509.CertPool) + WithClientCert(certFile, keyFile)` piecewise (mutually exclusive). Partial WithClientCert wiring caught at Connect.

**RPC:** `Request[Req, Resp](ctx, c, subject, req, timeout) (Resp, error)` + `Reply[Req, Resp](ctx, c, subject, queueGroup, handler) (*ReplySubscription, error)` for typed sync messaging; trace context propagated; codes `request_timeout` / `request_failed`.

**KV:** `EnsureKVBucket(ctx, KVConfig) nats.KeyValue` + `NewKV[T](c, bucket) *KV[T]` with `Get/Put/Update (CAS)/Delete/Raw()`; codes `kv_key_not_found` (NotFound) and `kv_op_failed` (Conflict for stale CAS rev). Replaces the kit-overview slot originally allocated to `jobs/`.

## `clients/httpc/`

Outbound HTTP client builder. `httpc.New(Config, ...Option) (*http.Client, error)` returns a stdlib `*http.Client` whose transport chain wraps the user-supplied or `http.DefaultTransport` with per-attempt `context.WithTimeout`, full-jitter exponential retry on transient failures (5xx, 429, 408, network errors — idempotent methods only; POST/PATCH never retry), and opt-in slog/Prometheus observability. `NewTransport(cfg, opts...)` returns the same chain unwrapped for users composing into their own client (otel, auth middleware).

Errors from validation map to `*errs.Error` (Codes: `httpc_invalid_timeout`, `httpc_invalid_max_retries`, `httpc_invalid_backoff`); runtime network errors and exhausted-retry responses pass through unchanged — drop-in stdlib semantics. `Retry-After` honoured (capped at `4 * BackoffMax`); body replay via `req.GetBody` (stdlib convention). `MaxRetries: -1` disables retries entirely; the zero value gets the default (3).

`WithBreaker(*breaker.Breaker)` inserts a circuit-breaker transport layer BETWEEN retry and base — each retry attempt independently consults the breaker, and the retry loop recognises `breaker.ErrOpen` as non-retryable so an open circuit bails immediately without burning the rest of the retry budget. Short-circuit surfaces as `*errs.Error{KindUnavailable, Code: "httpc_circuit_open"}` wrapping `ErrOpen` (so `errors.Is(err, breaker.ErrOpen)` holds and `errs.HTTP` yields 503). `WithBreakerFailureClassifier(func(*http.Response, error) bool)` overrides the default classifier (408/429/5xx + non-`context.Canceled` error → failure; non-transient 4xx → success). Sharing one `*breaker.Breaker` across multiple `*http.Client`s via repeated `WithBreaker(b)` is the supported pattern — that's how apimap's per-endpoint httpc-override clients share the per-upstream breaker.

`WithBulkhead(*bulkhead.Bulkhead)` inserts a concurrency-cap layer ABOVE breaker (so an open circuit does NOT occupy a slot — the breaker short-circuits before `base.RoundTrip` runs, and the deferred release fires on a slot that was never used) and BELOW retry (each retry attempt independently Acquires; a retry backoff sleep does NOT camp on a slot). Rejection surfaces as `*errs.Error{KindUnavailable, Code: "httpc_bulkhead_full"}` (wraps `bulkhead.ErrBulkheadFull`) when MaxConcurrent+MaxQueue is exhausted, or `KindTimeout, Code: "httpc_bulkhead_queue_timeout"` (wraps `bulkhead.ErrQueueTimeout`) when QueueTimeout fires. Retry treats `ErrBulkheadFull` as non-retryable (queue already saturated; piling retries makes it worse); `ErrQueueTimeout` IS retryable as a normal transient. Slot lifetime = one `base.RoundTrip` call — release fires on defer BEFORE the response body is read, so streaming bodies don't hold the slot. The package-level RNG used for backoff jitter was switched from a shared `*math/rand/v2.Rand` to the thread-safe top-level `math/rand/v2.Float64()` to fix a race the new concurrent bulkhead tests exposed.

**Retry-policy customization:** `WithRetryClassifier(func(req, resp, err) bool)` overrides the full default rule (status + method + network err) — honoured for BOTH success path and network-error path so callers can veto retry of `context.Canceled`-shaped errors. `WithRetryStatusCodes(...int)` atomically replaces just the transient-status set; `IsDefaultRetryableStatus(int) bool` exposes the kit default so a custom classifier can add to (rather than replace) it. `WithRetryOnNonIdempotent(true)` unconditionally permits POST/PATCH retry; `WithIdempotencyKeyHeader("Idempotency-Key")` is the Stripe-style opt-in — retry POST/PATCH ONLY when the outbound request carries the named header (caller-supplied idempotency-key contract with the upstream).

**Middleware chain:** `WithMiddleware(...func(http.RoundTripper) http.RoundTripper)` appends decorators ABOVE retry+metrics, BELOW the X-Request-ID stamp; applied in reverse so the first entry is the outermost. Each retry attempt sees user middleware as a separate RoundTrip (because retry lives below).

**Lifecycle hooks:** `WithBeforeRequest(func(req))` + `WithAfterResponse(func(req, resp, err, elapsed))` are short-API sugar over WithMiddleware; multiple calls — last wins; afterResponse fires even when the response is nil (network failure).

**Transport shortcuts:** `WithProxy(*url.URL)` + `WithDialer(...)` + `WithTLSConfig(*tls.Config)` populate a SHARED `*http.Transport` (each call mutates the same instance the previous shortcuts touched); explicit `WithBaseTransport` with a non-`*http.Transport` (otel wrapper) wins and the shortcuts no-op silently. Depends on `errs/`, `breaker/`, `bulkhead/`, and `prometheus/client_golang`; no fiber, no fasthttp.

## `clients/apimap/`

Declarative outbound HTTP layer. Upstream APIs are described in YAML (clients, endpoints, methods, paths, encoding/decoding, per-endpoint timeout/retry overrides, per-client auth); the engine validates everything at `Build` and returns a goroutine-safe `*Client` keyed by `<client>.<endpoint>`. Auth shapes (`type: basic|bearer|header|none`) resolve to a single header applied per-request; secrets come from `${ENV_VAR}` substituted into the YAML at Load. `apimap.Decode[T](ctx, c, name, Call) (T, error)` decodes 2xx response bodies; non-2xx maps to `*errs.Error` with `Kind` derived from status and a stable per-endpoint `Code` (e.g. `apimap_github_get_user_not_found`). Response context (status / url / truncated body) sits in `*errs.Error.Details`. `Exchange[Req, Resp]` adds typed request encoding. `Do(ctx, name, Call)` is the stdlib-style escape hatch returning `*http.Response` unwrapped.

Transport is `clients/httpc.New(...)` — one `*http.Client` per upstream by default, dedicated `*http.Client` per endpoint when any `timeout`/`max_retries`/`backoff_*` is overridden. Encoding modes: `json`/`form`/`raw`/`none`. Decoding modes: `json`/`raw`/`none`.

Optional per-client `breaker:` YAML block (`failure_threshold`, `minimum_requests`, `window_duration`, `window_size`, `open_interval`, `half_open_max_probes`) enables `breaker/` for the whole upstream — one `*breaker.Breaker` per `clients[].name`, threaded through every `*http.Client` built for that client INCLUDING endpoints with httpc-overrides (unit of failure = upstream, not endpoint, so a Stripe outage trips every Stripe endpoint together). Short-circuit surfaces as `*errs.Error{KindUnavailable, Code: "apimap_<client>_circuit_open"}` wrapping the underlying `breaker.ErrOpen` (the per-client Code suffix is generated at Build time). Build-time validation: `apimap_invalid_breaker` if the YAML block fails `breaker.New` (e.g. `minimum_requests < failure_threshold`). Breaker observability inherits from the engine's `WithLogger` / `WithMetrics` — `breaker_*{name=<client>}` series land on the same registry as `apimap_*` and `httpc_*`. Omitting the `breaker:` block disables the breaker for that client (opt-in feature; baseline behaviour unchanged).

Optional per-client `bulkhead:` YAML block (`max_concurrent` REQUIRED — there is no sensible default for "how much concurrency this upstream tolerates"; `max_queue` default 0 = fail-fast; `queue_timeout` default 0 = honour only caller ctx) enables `bulkhead/` for the whole upstream under the same per-client / endpoint-override-sharing rule as the breaker (a Stripe slowdown saturates every Stripe endpoint at once). Saturation surfaces as `*errs.Error{KindUnavailable, Code: "apimap_<client>_bulkhead_full"}` (wraps `bulkhead.ErrBulkheadFull`); queue-wait timeout: `*errs.Error{KindTimeout, Code: "apimap_<client>_bulkhead_queue_timeout"}` (wraps `bulkhead.ErrQueueTimeout`). Build-time validation: `apimap_invalid_bulkhead` if the YAML block fails `bulkhead.New` (e.g. `max_concurrent: 0`, `max_queue: -1`). Breaker and bulkhead can coexist on the same client (orthogonal failure modes — error rate vs concurrency); bulkhead wraps the breaker in the transport chain so an open circuit does not occupy a slot.

**httpc passthrough:** `apimap.WithHTTPCOptions(...httpc.Option)` engine-wide + `Engine.RegisterClientOptions(clientName, ...httpc.Option)` per-client unlock the full new httpc surface (retry classifier / WithRetryStatusCodes / WithIdempotencyKeyHeader / WithMiddleware / WithBeforeRequest / WithAfterResponse / WithProxy / WithDialer / WithTLSConfig). Per-client opts are appended AFTER the global slice at Build so client-specific opts refine rather than replace; unknown client names fail Build with `apimap_unknown_client`. WithLogger/WithMetrics/WithBaseTransport stay dedicated because apimap manages those internally — the metrics registry is deliberately NOT forwarded to httpc.

**apimap hooks:** `WithBeforeRequest(func(client, endpoint, *http.Request))` / `WithAfterResponse(func(client, endpoint, req, resp, err, elapsed))` see the kit-stable `<client>.<endpoint>` pair regardless of which *http.Client served the request — the endpoint name flows through a private context-key so a shared per-client transport still produces per-endpoint hook attribution. Multiple calls — last wins.

**Default Call:** `WithDefaultCall(Call)` engine-wide + `Engine.SetClientDefaultCall(clientName, Call)` per-client; defaults merge before the caller's Call (engine → client → caller, last wins on per-key conflict). Containers (Path / Query / Headers) merge by key; URL/Body take the last non-zero value.

**Mock mode:** `Engine.RegisterTransport(clientName, http.RoundTripper)` replaces the per-client base transport at Build with the supplied RoundTripper — the breaker / bulkhead / retry chain still wraps it so mock paths go through observability. Unknown names fail Build. Depends on `errs/`, `breaker/`, `bulkhead/`, `clients/httpc`, and `gopkg.in/yaml.v3`; no fiber, no fasthttp, no `db`/`auth`/`nats`.

## `clients/natsmap/`

Declarative NATS subscribers + publishers via YAML, symmetric to `clients/apimap`. Subscribers and publishers described in YAML (`subscribers.yaml`, `publishers.yaml`, or combined); Go code registers typed handlers + publishers by name (`natsmap.RegisterHandler[T]`, `natsmap.RegisterPublisher[T]`). `Engine.Build(ctx, *natsclient.Client) (*Runtime, error)` validates everything and opens subscriptions. `natsmap.Publish[T](ctx, rt, name, payload)` is the typed publish path; `*Runtime.Drain()` for graceful shutdown. Errors are `*errs.Error` with stable `natsmap_*` Code constants (`natsmap_no_entries`, `natsmap_handler_not_registered`, `natsmap_unknown_publisher`, etc.). Subscription dispatch reuses `clients/nats.SubscribeRaw` (added in the same PR) to share the ack/nak/term policy.

**natsclient passthrough:** `WithSubscribeOptions(...natsclient.SubscribeOption)` engine-wide + `Engine.RegisterSubscriberOptions(name, ...)` per-subscriber unlock the natsclient handler-resilience pack (WithErrorClassifier / WithAckProgress / WithPanicHandler) without writing a custom RawHandler. Unknown subscriber names fail Build with `natsmap_unknown_subscriber`.

**Hooks:** `WithBeforeDispatch(func(name, subject))` + `WithAfterDispatch(func(name, subject, err, elapsed))` for the subscriber path; `WithBeforePublish(ctx, name, subject, headers)` + `WithAfterPublish(ctx, name, subject, err, elapsed)` for the publisher path. beforePublish mutations land on the wire (use for ctx-aware trace headers).

**Default headers:** `WithDefaultPublishHeaders(map)` engine-wide; layering is defaults → publisher YAML static → per-call (last wins per-key); X-Request-ID from ctx still auto-injects unless explicit.

**Metrics:** `WithMetrics(reg)` now wires natsmap-owned `natsmap_handlers_total{name,outcome}` + `natsmap_handler_duration_seconds{name}` + `natsmap_publishes_total{name,outcome}`. Cardinality bounded by YAML-declared `name`; subscription-level series stay on clients/nats.

**Mock mode:** `RegisterMockHandler[T](e, name, fn)` + `DispatchMock[T](ctx, rt, name, payload, headers)` for unit-testing services that wire natsmap without spinning a NATS testcontainer. Mock subscribers skip every NATS-side wiring at Build; `DispatchMock` fires the registered fn synchronously on the caller's goroutine and BYPASSES before/after hooks by design (test-time path). Build now tolerates a nil `*natsclient.Client` when every subscriber is a mock and no publisher is declared; publishers in that mode install error-stubs so accidental Publish calls surface loud.

**Batched handler:** `RegisterBatchedHandler[T any](e, name, func(ctx, []natsclient.Msg[T]) error)`. YAML subscribers gain `batch_size: N` (opts into Pull mode) and `batch_interval: <duration>` (max wait between fetches; default 1s). At Build, batched-mode subscribers open a `nats.JetStreamContext.PullSubscribe` and run a fetch loop: `Fetch(batch_size, MaxWait=batch_interval)` → decode each → call the typed batched handler with the slice → on nil return Ack every msg, on err return Nak every msg → JetStream redelivers the whole batch. Mode mismatches caught at Build with `natsmap_batch_handler_required` / `natsmap_regular_handler_required`. Decode failures Term the offending msg (poison-pill suppression) and remove it from the live batch; remaining decoded msgs still proceed. Runtime.Drain stops the fetch goroutine + unsubscribes. Implementation in `clients/natsmap/batched.go`. urlshort visit_counter migrated to this — Handle is `func(ctx, []natsclient.Msg[LinkVisited]) error` and the kit handles ack atomicity.

Depends on `errs/`, `clients/nats/`, `gopkg.in/yaml.v3`. Tests use `testcontainers-go/modules/nats`. The matching `service/` integration auto-builds the Runtime when `NATSMAP_SUBSCRIBERS_PATH` or `NATSMAP_PUBLISHERS_PATH` is set; see `docs/packages/service.md`.

## `clients/redis/`

Redis client wrapper. `redisclient.Connect(ctx, Config, opts...) (*Client, error)` parses a `redis://` URL, applies optional `WithRedisOptions(fn)` mutator, retries the initial PING with exponential backoff (same retry budget pattern as db/nats). `Client` now wraps `redis.UniversalClient` internally so the same kit type covers single-node, cluster, and sentinel deployments.

**Topology entry points:** `Connect` (single), `ConnectCluster(ctx, ClusterConfig, opts...)` (redis.NewClusterClient), `ConnectSentinel(ctx, SentinelConfig, opts...)` (redis.NewFailoverClient — HA failover).

**Cross-mode API:** `Client.Universal() redis.UniversalClient` returns the wrapped handle regardless of mode; `Client.Redis() *redis.Client` stays for single-mode back-compat (returns nil under cluster/sentinel); `Client.Mode() Mode` reports the topology. Per-mode mutators: `WithClusterOptions(func(*redis.ClusterOptions))`, `WithSentinelOptions(func(*redis.FailoverOptions))` paralleling `WithRedisOptions`.

Opt-in observability via `WithLogger`/`WithMetrics` installs a go-redis Hook that records every command (kit-issued AND user-issued) — collectors: `redis_commands_total{cmd,outcome}`, `redis_command_duration_seconds{cmd}`, `redis_pool_size_total{state}` (cluster-mode aggregates across shards), and `redis_connection_status` (1 after Connect ping success, 0 after Close — symmetric with `nats_connection_status`). `redis.Nil` is intentionally NOT counted as `outcome=error` — it's the operational "key not found" signal.

**Composability:** `WithHook(redis.Hook)` appends user hooks AFTER the kit observability hook (multiple calls accumulate).

**Resilience:** `WithDefaultTimeout(d)` wraps every command's ctx via `context.WithTimeout(d)` when the caller has no deadline (explicit caller deadline always wins) — defence-in-depth against runaway commands. `WithBreaker(*breaker.Breaker)` routes every command through `breaker.Execute`; `redis.Nil` counts as success so the breaker doesn't trip on "key not found", any other err counts as failure; open-state surfaces as `*errs.Error{KindUnavailable, Code: "redis_circuit_open"}` wrapping `breaker.ErrOpen` (`errors.Is(err, breaker.ErrOpen)` holds).

Service auto-wires when `service.Config.Redis.URL` is set; result is `svc.Redis`. Close is nil-safe + idempotent. Package name is `redisclient` to avoid colliding with go-redis's own `redis` package (same convention as `natsclient`).

## `clients/cache/`

Typed Redis-backed read-through cache. `cache.New[T](rdb redis.UniversalClient, cfg) (*Redis[T], error)` returns a generic cache over any encodable value. Accepts `redis.UniversalClient` so single-node, cluster, and sentinel deployments work without code changes; `cache.For[T](rc *redisclient.Client, prefix) *Redis[T]` is the kit-aware shortcut that threads through `rc.Universal()`.

API: `Get(ctx, key) Lookup[T]{Value, NotFound}` tri-state — positive hit / negative hit / miss; `Set(ctx, key, T)`; `SetNotFound(ctx, key)` stores a single-byte sentinel for the negative-cache window (default 60s TTL, absorbs scanner traffic at zero DB cost); `Invalidate(ctx, key)`; `InvalidatePrefix(ctx, partial)` walks via SCAN + pipelined DEL (no KEYS; cluster-mode caveat — SCAN runs per-shard, pin tenant keys via hashtag for full coverage).

`GetOrLoad(ctx, key, LoaderFn[T])` is the read-through helper with an internal `singleflight.Group` collapsing concurrent requests on the same key into one loader invocation. `LoaderFn[T] = func(ctx, key) (T, bool, error)` — `(val, true, nil)` → positive cache, `(zero, false, nil)` → SetNotFound, `(zero, false, err)` → caller without poisoning.

Best-effort error policy — Redis transport failures are logged + swallowed (Get returns a miss, writes no-op); encode/decode failures likewise. `(*Redis[T])(nil)` is safe on every method, letting callers thread a cache reference unconditionally.

KeyPrefix is required at construction so a shared Redis instance can host multiple caches without key collision. `Config.Codec` (pluggable `Marshal/Unmarshal` interface; JSON default) lets callers swap to msgpack / protobuf. `Config.TTLJitter` (0..1 fraction; default 0) applies ±jitter to PositiveTTL/NegativeTTL on every Set — defends popular keys against sync-expiry storms.

`Config.Name + Config.MetricsReg` register `cache_operations_total{name,operation,outcome}` (operations: get/set/set_not_found/invalidate/invalidate_prefix; outcomes: hit/miss/negative/ok/error) and `cache_operation_duration_seconds{name,operation}` — Name required when MetricsReg is set. urlshort migrated to this; see `internal/links/service.go`.

## `clients/webhooks/`

Outbound + inbound HTTP webhooks. `Subscription` / `Delivery` types + `SubscriptionStore` / `DeliveryStore` interfaces; outbound `Signer` produces Stripe-style `X-Webhook-Signature: t=<unix>,v1=<hmac-sha256>`; `Fanout` turns one domain Event into N deliveries (idempotent via UNIQUE `(subscription_id, event_id)`); `Worker` drains pending rows with per-target exponential backoff (default `Initial=1s`, `Max=1h`, `MaxAttempts=8`) and per-target DLQ on exhausted retries OR fatal 4xx response (4xx ≠ 408/429 goes straight to DLQ).

Postgres backend in `clients/webhooks/storepg` with AES-256-GCM secret-at-rest — `WEBHOOKS_SECRET_KEY` (32 bytes base64), version-prefixed ciphertext for future ratchet. `RetentionWorker` deletes `delivered` rows older than TTL (default 30d); DLQ rows are NEVER auto-deleted (operator inspects). Inbound verification in `fibermap/webhookguard` with `Verifier` interface + ready presets in `clients/webhooks/verifiers/` (`GenericHMAC` configurable scheme; `GitHub` preset for `X-Hub-Signature-256`).

`service.WithWebhooks(WebhooksConfig{SubStore, DeliveryStore, StartWorker, StartFanout})` wires Worker into the lifecycle and drains it via `OnShutdown` (LIFO so it stops BEFORE NATS/DB teardown); `Service.WebhooksFanout` is exposed for the caller to plug into their `WithNATSMapRegistration` handler — kit stays neutral to which event types fan out. Stores are user-built (not auto-wired) so secret-encryption key wiring stays explicit. Errors as `*errs.Error` with stable `webhooks_*` / `webhook_signature_invalid` codes.

**Worker resilience pack:** `WorkerConfig.BreakerFactory(subID) *breaker.Breaker` builds per-subscription breakers cached in sync.Map — open-state short-circuits as retryable so known-down endpoints don't burn in-flight slots. `WorkerConfig.RetryClassifier(*http.Response, error) Outcome` overrides the default 2xx/408/429/5xx mapping; `Outcome` type + `DefaultClassifier` building block exported. `WorkerConfig.OnAttempt(d, resp, err, outcome, elapsed)` + `WorkerConfig.OnDLQ(d, status, errMsg)` are observability hooks (panic-safe). `WorkerConfig.AttemptTimeout` replaces hardcoded 30s. Panic recovery in attempt goroutine — slot released + scheduled retry; production-grade against panicking RoundTripper. `WorkerConfig.SignerFunc` swaps Stripe-style HMAC for any caller scheme. `WorkerConfig.DefaultContentType` overrides hardcoded application/json (Delivery.Headers still win). `WorkerConfig.Propagator propagation.TextMapPropagator` injects W3C TraceContext (traceparent/tracestate) — same shape as clients/nats publish. `NewChecker(store, name) *Checker` is the /readyz adapter — pings DeliveryStore.Claim(ctx, 0); same shape as clients/nats.Checker / clients/redis.Checker.
