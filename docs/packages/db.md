# db

pgx-pool-обёртка, мульти-реплики, helpers, testcontainers.

## `db/`

pgx-based pool wrapper. `*db.DB` from `db.Connect(ctx, cfg, opts...)` exposes `Query/QueryRow/Exec` (errors mapped to `*errs.Error` via `mapPgxErr`), `Tx(ctx, fn)` for functional transactions (nested calls open savepoints via `pgx.Tx.Begin`), `Healthcheck`, and a `Pool()` escape hatch. `TxWithOpts(ctx, TxOpts{IsoLevel, AccessMode, DeferrableMode}, fn)` is the explicit form for SERIALIZABLE / READ ONLY / DEFERRABLE transactions; `Tx` is the shortcut for `TxOpts{}` (READ COMMITTED, read-write). `TxRetry(ctx, fn, opts...)` wraps `TxWithOpts` with auto-retry on SQLSTATE 40001/40P01 (exp-backoff + ±25% jitter, defaults `MaxAttempts=3 / 5ms / 100ms`); options `WithTxRetryMaxAttempts`, `WithTxRetryBackoff`, `WithTxRetryClassifier`, `WithTxRetryOpts(TxOpts)`. Helper `IsRetryableTxConflict(err)` walks via `errors.As` so wrapped `*errs.Error` still classifies. `WithDefaultStatementTimeout(d)` injects server-side `SET statement_timeout` via `AfterConnect` so runaway queries die on the server even when caller ctx is gone; `WithConnInit(fn ConnInitFn)` is the generic per-connection hook chained after the timeout setter, accumulates across calls — used for `SET application_name`, `SET search_path`, `SET ROLE`, prepared-stmt warming. `HealthcheckRead(ctx)` pings every configured read pool (no-op when no read replica) — returns the first failure with the offending pool name in the error message so operators see which standby is down; `Healthcheck` stays primary-only so /readyz can fail on primary loss while standby loss is a diagnostic signal. `CopyFrom` (on `*DB` and `*Tx`) is the thin COPY-protocol bulk-insert wrapper, errors funneled through the same `mapPgxErr`.

**Multi-replica routing:** `Config.ReadURLs []string` (env `DB_READ_URLS=a,b,c`) opens one pgxpool per URL and rounds `ReadQuery` / `ReadQueryRow` across them via `Config.ReadStrategy` (`round_robin` default — atomic counter, no locks; `random` — math/rand/v2). `ReadURLs` takes precedence over `HasReadReplica`. Each URL gets `target_session_attrs=standby` injected unless it already carries the param (analytics replicas can pass `any`). `(d) ReadPools() []ReadPoolInfo{Name, Pool, Healthy, LagSeconds}` exposes every pool with its stable `standby`/`standby-N` name (matches the `pool=` metric label) AND its current health / lag snapshot (kit-default Healthy=true, LagSeconds=0 when polling isn't wired); `(d) HasReadReplica() bool` reports whether at least one replica is wired; `(d) ReadPool()` keeps back-compat returning the first entry. `db.ReadFromPrimary(ctx) ctx` is the read-your-writes ctx-marker — wraps any ctx so subsequent `ReadQuery` / `ReadQueryRow` calls land on the primary instead of a standby (deterministic no-op on a no-replica deployment).

**Replication observability:** `(d) ReplicationLag(ctx) []ReplicaLagInfo{PoolName, LagSeconds, Healthy, Err}` queries `EXTRACT(EPOCH FROM (now() - pg_last_xact_replay_timestamp()))` per replica — primary nodes return NULL → kit treats as `LagSeconds=0, Healthy=true` so the metric stays sensible; empty slice when no replica is configured. `WithReplicaLagPolling(interval, threshold)` spawns a background goroutine that polls every replica at `interval`, updates the `db_replica_lag_seconds{pool}` gauge (`-1` on probe failure so dashboards can distinguish "unknown" from "caught up"), and emits a per-replica WARN through `WithLogger` when lag exceeds `threshold > 0`; the goroutine exits on `Close()` / `Drain()`.

**Smart read-routing** via `WithReadLagBudget(d)` — replicas with tracked lag > `d` OR unhealthy probes are skipped; ReadQuery falls back to primary when every replica is filtered out. Pairs with `WithReplicaLagPolling` (lag is only tracked when the poller is running).

**Env-driven multi-replica setup:** `Config.LagBudget` (env `DB_LAG_BUDGET`), `Config.LagPollInterval` (`DB_LAG_POLL_INTERVAL`), `Config.LagPollThreshold` (`DB_LAG_POLL_THRESHOLD`) — db.Connect auto-applies `WithReplicaLagPolling` + `WithReadLagBudget` from these Config fields BEFORE user-supplied opts (so explicit programmatic configuration overrides via last-write-wins). The full env-driven recipe: `DB_READ_URLS=…,…` + `DB_LAG_POLL_INTERVAL=10s` + `DB_LAG_POLL_THRESHOLD=30s` + `DB_LAG_BUDGET=5s` — service.New picks this up automatically. Freshly-started replicas (lag = -1, no probe yet) are treated as in-budget so the kit doesn't surprise callers during the first poll interval.

**Query helpers** (package-level, take `Querier`): `Exists(ctx, q, sql, args)` wraps SELECT EXISTS, `Count(ctx, q, sql, args)` wraps SELECT count(*) FROM (sub), `Pluck[T](ctx, q, sql, args) ([]T, error)` for single-column → slice (generic), `Get[T](ctx, q, sql, args) (T, error)` single-row single-column; all nil-safe (nil Querier → `KindValidation`, Code `db_nil_querier`); `NotFound(err) bool` is the "no rows" classifier shortcut.

**Batch:** `NewBatch().Queue(sql, args).Queue(...)` collects N statements; `(d).SendBatch(ctx, b)` or `(tx).SendBatch(ctx, b)` ships them in one round-trip via pgx extended-query protocol and returns `*BatchResults` (must Close); iterate via Exec/Query/QueryRow in queue order, over-iteration → `Code: db_batch_overrun`. For thousands of rows use CopyFrom (Batch has per-statement protocol overhead).

**Query-name tagging:** `WithQueryName(ctx, name) ctx` tags ctx so the kit's pgx tracer emits `db_query_duration_seconds{name="<name>", outcome="..."}` (empty `name=""` for untagged queries — that label is now always present in the histogram). Cardinality is the operator's responsibility — never use user-controlled input; kit makes no attempt to bound it.

**Connection pinning:** `(d).WithConn(ctx, fn func(*pgxpool.Conn) error)` acquires one primary-pool conn, runs fn pinned to it, releases on return — for temp tables / prepared stmts / cursor FETCH / SET LOCAL outside a tx; `(d).WithReadConn(ctx, fn)` is the read-replica variant honouring the same routing as ReadQuery (including ReadFromPrimary marker). fn errors surface verbatim; Acquire/Release failure maps to KindUnavailable.

**Pool snapshot:** `(d).Stats() Stats{Primary PoolStats, Replicas []PoolStats, HasReplicas bool}` returns a single-call pgxpool stats + replica health/lag projection for /admin endpoints — no Prometheus scrape needed. PoolStats carries Name (matches `pool=` label), Acquired/Idle/Max/Total, and Healthy + LagSeconds for replicas (zero on primary).

**Keyset pagination:** `Paginate[T,K](ctx, q, sql, limit, scan, args...) (Page[T]{Items, NextCursor}, error)` is the cursor-based pagination helper — caller writes the SQL (`WHERE key > $cursor ORDER BY key LIMIT N+1`), the helper handles the +1 sentinel + opaque cursor (base64 JSON via `EncodeCursor[K]` / `DecodeCursor[K]`). Malformed cursor → `KindValidation, Code: db_cursor_invalid`.

**Bulk UPDATE:** `NewBulkUpdate(table, keyCol).Columns(...).Add(key, vals...).Exec(ctx, q) (int64, error)` builds `UPDATE table SET ... FROM (VALUES (...)) AS t(...) WHERE table.key = t.key` for single-round-trip multi-row updates; stable Codes (`db_bulk_no_table`, `db_bulk_no_key`, `db_bulk_no_columns`, `db_bulk_row_arity`). Empty bulk is no-op. For thousands of rows prefer CopyFrom into a temp table + single UPDATE — VALUES list grows linearly.

Observability is opt-in via `WithLogger(*slog.Logger)`, `WithMetrics(prometheus.Registerer)`, `WithSlowQueryThreshold(time.Duration)`; counter `db_tx_retries_total` increments once per retry attempt; gauge `db_replica_lag_seconds{pool}` populated by lag polling; counter `db_replica_skipped_total{pool, reason=unhealthy|over_budget}` + `db_replica_fallback_total` populated by smart routing. No fiber dep. Tests use `testcontainers-go/modules/postgres` — Docker required.

## `db/sqb/`

Opt-in squirrel wrapper preconfigured for `$N` placeholders. `sqb.Builder` for query construction, `sqb.Query`/`sqb.Exec` to run builders against any `db.Querier`. Depends on `db/` (one-way; core `db/` does NOT import `sqb`).

## `db/testdb/`

Testing-only helpers that spin Postgres testcontainers and hand back ready *db.DB. `testdb.Spin(t, ...Option) *db.DB` boots `postgres:16-alpine` once per test binary, reuses the container across calls, and isolates each call via per-test `test_<hex>` schema with `SET search_path` pinned (`MaxConns: 1` so the search_path persists). Lifecycle (Close + Terminate) registered via `t.Cleanup`; skips under `go test -short`. `WithFreshPerTest()` opts out of reuse — slower but guaranteed isolation for tests that mutate global server state.

`testdb.SpinCluster(t, replicas, ...ClusterOption) *Cluster` is the multi-node helper: it boots `bitnamilegacy/postgresql:16` containers on a dedicated docker network (Bitnami's env-driven `POSTGRESQL_REPLICATION_MODE=master|slave` removes the pg_basebackup dance vanilla `postgres:` would require), with the primary aliased "primary" and standbys "standby-N", returning `Cluster{Primary, Replicas []*db.DB, Multi *db.DB}` — Multi is a single *db.DB pre-wired with `Config.ReadURLs` spanning every replica so production multi-replica routing code Just Works in tests. `Cluster.WaitForReplication(ctx)` polls `pg_current_wal_lsn() - pg_last_wal_replay_lsn()` every 50ms until every replica is caught up or ctx expires — use immediately after a write before reading from any standby. Cluster bootstraps are ALWAYS per-call (cross-test state in a streaming pair is hard to clean cleanly); single-node `Spin` is reused. Replaces ~30 lines of `TestMain + initContainer + Connect` boilerplate that currently duplicates across ~15 kit subpackages.

Options: `WithImage` / `WithClusterImage`, `WithDatabase`, `WithCredentials`, `WithReplicationCredentials` (cluster), `WithStartupTimeout` (default 120s), `WithMaxConns` (default 4; Spin overrides to 1), `WithConfigKVs map[string]string` (extra `postgresql.conf` overrides, passed as `-c key=value` to the cluster primary).
